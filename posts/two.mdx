> 호이스팅을 '**끌어올려지는거**' 라고만 생각하셨던분들 꼭 이 글을 봐주세요!

# 1. 호이스팅이란?

JavaScript는 코드 실행 시 변수 및 함수 선언을 어떻게 처리하는지에 대한 개념인 "호이스팅"을 가지고 있습니다. 호이스팅은 코드 실행 전에 JavaScript 엔진에 의해 처리되는 프로세스로, 코드의 흐름을 이해하고 예측하는 데 중요한 역할을 합니다.

# 2. 변수 호이스팅

변수 선언은 코드의 맨 위로 끌어올려집니다. 이것은 변수를 선언하기 전에 사용할 수 있다는 의미는 아닙니다. 변수의 선언은 끌어올려지지만 할당은 원래 위치에 남아 있습니다.

```javascript
console.log(x); // undefined
var x = 5;
```

위의 코드는 다음과 같이 해석됩니다.

```javascript
var x;
console.log(x); // undefined
x = 5;
```

# 3. 함수 호이스팅

함수 선언도 변수와 마찬가지로 **호이스팅** 됩니다. `함수선언`은 코드의 맨 위로 올라가며 함수를 선언하기 전에 호출할 수 있습니다.

```javascript
sayHello(); // "Hello, world!"

function sayHello() {
  console.log("Hello, world!");
}
```

함수 호이스팅은 변수에 할당된 함수 표현식에서는 동작하지 않습니다.

```javascript
sayHello(); // TypeError: sayHello is not a function

var sayHello = function () {
  console.log("Hello, world!");
};
```

# 4. 변수와 함수 호이스팅의 주의사항

호이스팅은 JavaScript의 동작 방식을 이해하는 데 도움이 되지만 주의해야 할 몇 가지 중요한 사항이 있습니다.

1. 변수와 함수가 선언되기 전에 사용되더라도 변수는 undefined로 초기화되고 함수는 호출됩니다. 이로 인해 예기치 않은 동작이 발생할 수 있으므로 주의해야 합니다.

2. 호이스팅은 변수 및 함수 선언을 코드의 맨 위로 올려놓지만, 변수 초기화 및 함수 할당은 원래 위치에서 수행됩니다.

3. 함수 표현식은 호이스팅되지 않으므로 함수 표현식을 사용할 때는 해당 위치에서 선언한 후 호출해야 합니다.

호이스팅은 JavaScript의 중요한 특성 중 하나이며 코드를 이해하고 디버깅하는 데 도움을 줄 수 있습니다. 하지만 호이스팅에 대한 이해가 부족하면 예상치 못한 결과를 낳을 수 있으므로 주의 깊게 사용해야 합니다.
